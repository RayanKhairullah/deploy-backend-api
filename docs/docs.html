API Documentation untuk Frontend

Berikut adalah dokumentasi untuk penggunaan endpoint API.

Endpoint API

Headers: Content-Type: application/json (digunakan untuk endpoint register, verify-email, dan login)

Register

URL: POST /register

Request example:

{
  "username": "username",
  "email": "emailemail@gmail.com",
  "password": "securepassword"
}

Response:

{
  "status": "success",
  "message": "Registrasi berhasil, cek email untuk verifikasi"
}

Verify Email

URL: POST /verify-email

Request example:

{
  "email": "emailemail@gmail.com",
  "code": "pfoGvpK1ba"
}

Response:

{
  "status": "success",
  "message": "Email berhasil diverifikasi"
}

Login

URL: POST /login

Request example:

{
  "email": "emailbudi@gmail.com",
  "password": "securepassword"
}

Response:

{
  "status": "success",
  "message": "Login berhasil",
  "data": {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

Create Expense

URL: POST /expenses

Request example:

{
  "category": "Food",
  "uangMasuk": 0,
  "uangKeluar": 50000,
  "uangAkhir": 950000,
  "description": "Lunch with friends",
  "transaction_date": "2025-03-28"
}

Response:

{
  "status": "success",
  "message": "Expense berhasil ditambahkan",
  "data": {
      "expenseid": "random11char"
  }
}

Get All Expenses

URL: GET /expenses

Response:

{
  "status": "success",
  "data": {
      "expenses": [
          {
              "expenseid": "random11char",
              "category": "Food",
              "uangmasuk": 0,
              "uangkeluar": 50000,
              "uangakhir": 950000,
              "description": "Lunch with friends",
              "transaction_date": "2025-03-28",
              "createdAt": "2025-03-28T12:00:00.000Z",
              "updatedAt": "2025-03-28T12:00:00.000Z"
          }
      ]
  }
}

Get Expense by ID

URL: GET /expenses/{expenseid}

Response (Jika Ditemukan):

{
  "status": "success",
  "data": {
      "expense": {
          "expenseid": "random11char",
          "category": "Food",
          "uangmasuk": 0,
          "uangkeluar": 50000,
          "uangakhir": 950000,
          "description": "Lunch with friends",
          "transaction_date": "2025-03-28",
          "createdAt": "2025-03-28T12:00:00.000Z",
          "updatedAt": "2025-03-28T12:00:00.000Z"
      }
  }
}

Response (Jika Tidak Ditemukan):

{
  "status": "fail",
  "message": "Expense tidak ditemukan"
}

Update Expense by ID

URL: PUT /expenses/{expenseid}

Request example:

{
  "category": "Transport",
  "uangMasuk": 0,
  "uangKeluar": 20000,
  "uangAkhir": 930000,
  "description": "Taxi fare",
  "transaction_date": "2025-03-28"
}

Response (Jika Berhasil):

{
  "status": "success",
  "message": "Expense berhasil diperbarui"
}

Response (Jika Tidak Ditemukan):

{
  "status": "fail",
  "message": "Expense gagal diperbarui. Id tidak ditemukan"
}

Delete Expense by ID

URL: DELETE /expenses/{expenseid}

Response (Jika Berhasil):

{
  "status": "success",
  "message": "Expense berhasil dihapus"
}

Response (Jika Tidak Ditemukan):

{
  "status": "fail",
  "message": "Expense gagal dihapus. Id tidak ditemukan"
}

Me

URL: GET /me

Response (Jika Berhasil):

{
  "status": "success",
  "message": "Berhasil mendapatkan data user",
  "data": {
      "id": 1,
      "username": "user",
      "email": "emailbudi@gmail.com",
      "verified": true,
      "createdAt": "2025-04-15T06:38:50.433Z"
  }
}

Logout

URL: POST /logout

Response (Jika Berhasil):

{
  "status": "success",
  "message": "Logout berhasil"
}

Response (Jika Gagal):

{
  "status": "fail",
  "statusCode": 401,
  "message": "Token tidak valid atau kadaluarsa"
}
